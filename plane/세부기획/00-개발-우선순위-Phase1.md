# 개발 우선순위 - Phase 1 (즉시 개발 가능)

> **목표**: BD/파트너 섭외 없이 즉시 개발 가능한 플랫폼 기능 중심

---

## 📊 기능 분류 기준

### ✅ Phase 1: 즉시 개발 가능
- **소프트웨어만으로 구현 가능**
- **외부 파트너/업체 섭외 불필요**
- **사용자 데이터 기반 자동화**
- **정보 제공 및 기록 관리**

### ⏸️ Phase 2: BD 완료 후 개발
- **실제 서비스 제공업체 필요** (청소업체, 기사, 병원 등)
- **IoT 하드웨어 연동 필요**
- **물리적 배송/방문 서비스**
- **제3자 업체와의 계약/정산 시스템**

## 🎯 Phase 1 개발 우선순위

### 1순위: 헬스케어 - 자가 관리 기능 ⭐⭐⭐

#### 1-1. 건강 대시보드
**즉시 개발 가능 ✅**
- 건강 점수 계산 로직
- 오늘 할 일 목록
- 건강 기록 타임라인
- 주간 활동 추적

**개발 범위:**
```typescript
// 모두 프론트엔드 + DB로 구현 가능
- 건강 데이터 입력 폼
- 점수 계산 알고리즘
- 데이터 시각화 (차트, 그래프)
- 알림 시스템
```

#### 1-2. 복약 관리 시스템
**즉시 개발 가능 ✅**
- 약 등록 (수동 입력)
- 복용 시간 알림
- 복용 기록 관리
- 약 재고 추적 (자동 계산)
- 복약 캘린더

**Phase 2로 미루기 ⏸️**
- 약 사진 AI 인식 (OCR)
- 약물 상호작용 체크 (약물 DB 구축 필요)

**개발 범위:**
```typescript
// Phase 1
- 약 수동 등록 (이름, 용량, 복용시간)
- 푸시 알림 (복용 시간)
- 복용 완료 체크
- 재고 자동 계산
- 가족 알림 (미복용 시)

// Phase 2
- 약봉지 사진 촬영 → AI 자동 인식
- 성분 분석 및 충돌 체크
```

#### 1-3. 건강 기록 관리
**즉시 개발 가능 ✅**
- 혈압, 혈당, 체중, 체온 수동 입력
- 기록 타임라인
- 그래프/차트 시각화
- 가족 공유 기능
- PDF 내보내기

**Phase 2로 미루기 ⏸️**
- 스마트워치 자동 연동 (IoT)
- 병원 진료 기록 자동 연동 (병원 API)

#### 1-4. 응급 상황 대응
**즉시 개발 가능 ✅**
- SOS 버튼 (3초 길게 누르기)
- 가족 자동 알림 (SMS/푸시)
- 실시간 위치 공유
- 응급 정보 카드

**Phase 2로 미루기 ⏸️**
- 119 자동 다이얼 (통신사 협의 필요)
- 낙상 감지 (스마트워치 연동)

#### 1-5. AI 건강 어시스턴트
**즉시 개발 가능 ✅**
- 증상 입력 → 진료과 추천 (룰 베이스)
- 건강 상담 챗봇 (GPT API 활용)
- 주간 건강 리포트 자동 생성
- 예측 케어 알림 (패턴 분석)

**개발 범위:**
```typescript
// GPT API 활용
- "무릎이 아파요" → "정형외과 추천"
- "혈압 150/95" → "고혈압 1기, 병원 방문 권장"
- 건강 데이터 분석 → 주간 리포트 생성
```

### 2순위: 헬스케어 - 정보 제공 기능 ⭐⭐

#### 2-1. 병원 찾기 & 정보
**즉시 개발 가능 ✅**
- 증상 기반 병원 검색
- 병원 정보 DB (공공 데이터 활용)
- 위치 기반 검색
- 리뷰 시스템 (사용자 작성)
- 예상 진료비 정보

**Phase 2로 미루기 ⏸️**
- 실시간 병원 예약 연동
- 병원 동행 서비스 (돌봄 인력 섭외 필요)

**개발 범위:**
```typescript
// Phase 1: 정보 제공 플랫폼
- 병원 DB 구축 (공공데이터 API 활용)
- 검색 및 필터링
- 리뷰 시스템
- 즐겨찾기/북마크
- 병원 전화 연결 (tel: 링크)

// Phase 2: 실제 예약 시스템
- 병원 API 연동
- 예약 시스템
- 동행 서비스 매칭
```

#### 2-2. 병원 예약 관리 (자체 캘린더)
**즉시 개발 가능 ✅**
- 병원 예약 일정 수동 입력
- 캘린더 관리
- 예약 알림 (1일 전, 1시간 전)
- 가족 공유
- 준비물 체크리스트

**개발 범위:**
```typescript
// 사용자가 직접 병원 예약 후 앱에 기록
- 병원 이름, 날짜, 시간 입력
- 증상 메모
- 알림 설정
- 가족에게 공유
```

## 💰 Phase 1 예상 비용

### 개발 비용
- **인력**: 풀스택 개발자 2명 (12주)
- **디자이너**: UI/UX 디자이너 1명 (4주)

### 운영 비용
- **서버**: Vercel/Supabase (월 ~$50)
- **푸시 알림**: Firebase (무료)
- **GPT API**: OpenAI (월 ~$100)
- **지도 API**: 카카오맵/네이버맵 (무료 tier)
- **SMS**: 알림톡/문자 (월 ~$50)

### 데이터 구축
- **병원 DB**: 공공데이터포털 (무료)
- **식품 영양 DB**: 식품의약품안전처 (무료)

## 🎯 핵심 전략

### Phase 1의 목표
> **"시니어가 혼자서도 건강을 관리할 수 있는 디지털 건강 수첩"**

1. **복약 관리**: 절대 놓치지 않도록
2. **건강 기록**: 병원 갈 때 의사에게 보여줄 수 있게
3. **가족 안심**: 부모님이 잘 지내시는지 확인
4. **AI 도우미**: 궁금한 거 물어보기

### Phase 2의 목표
> **"필요한 모든 서비스를 한 곳에서"**

1. **병원 예약**: 클릭 한 번에 예약 완료
2. **생활 서비스**: 청소, 수리, 배달 통합
3. **이동 지원**: 시니어 전용 교통
4. **스마트홈**: IoT로 안전하게
