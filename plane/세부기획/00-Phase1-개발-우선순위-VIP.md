# Phase 1 개발 우선순위 - VIP 홈헬스케어 플랫폼
## (바야다 전략 기반)

> **목표**: 바야다 핵심 사업(방문 간호/요양)의 디지털 전환을 즉시 개발 가능한 기능부터 시작

---

## 📊 플랫폼 재정의

### 기존 기획 vs 바야다 VIP 플랫폼

| 구분 | 기존 (시니어 일반) | 바야다 VIP |
|-----|-------------------|------------|
| **타겟** | 건강한 시니어 | 중증 질환/장애 환자 |
| **핵심 가치** | 건강 관리 편의성 | 전문 홈헬스케어 |
| **서비스** | 복약 알림, 병원 찾기 | 방문 간호/요양 |
| **파트너십** | 일반 업체 | 에버엑스, K-DOC, DKSH |
| **수익 모델** | 플랫폼 수수료 | 장기요양보험 + 수수료 |
| **차별성** | 시니어 친화 UX | 간호사 중심 전문 케어 |

## 🎯 Phase 1 개발 우선순위 (6개월)

### 1순위: 장기요양 인정 신청 대행 시스템 ⭐⭐⭐
**비즈니스 핵심 ✅**

#### 개발 이유
- **고객 유입의 시작점**: 무료 대행 서비스로 리드 확보
- **경쟁 차별화**: 대부분 요양 업체가 제공하지 않는 서비스
- **즉시 개발 가능**: 외부 연동 불필요, 폼 작성 + 제출 시스템

#### 개발 범위
```typescript
[웹 폼]
  ↓
[신청 정보 입력]
  ↓
[서류 업로드]
  ↓
[바야다 내부 검토]
  ↓
[국민건강보험공단 대행 제출]
  ↓
[진행 상황 알림]
```

#### 핵심 기능
- 신청인 정보 입력 폼
- 건강 상태 체크리스트
- 서류 업로드 (신분증, 의사 소견서)
- 신청 진행 상황 트래킹
- 등급 판정 결과 알림
- 가족 앱 동기화

#### 예상 소요 시간
- **2주** (디자인 포함)

### 2순위: 방문 간호/요양 매칭 시스템 ⭐⭐⭐
**비즈니스 핵심 ✅**

#### 개발 이유
- **바야다 핵심 비즈니스**: 방문 간호/요양이 주 수익원
- **즉시 개발 가능**: 매칭 알고리즘 + 일정 관리
- **MVP 가능**: 자동 매칭 없이 관리자 수동 배정으로 시작

#### 개발 범위 (MVP)

##### 2-1. 방문 간호 요청
```typescript
[방문간호지시서 업로드]
  ↓
[건강 상태 입력]
  ↓
[간호사 수동 배정] ← 관리자
  ↓
[일정 조율]
  ↓
[방문 확정]
```

##### 2-2. 방문 요양 요청
```typescript
[장기요양 인정서 확인]
  ↓
[건강 상태 입력]
  ↓
[초기 방문 일정 잡기] (간호사 + 요양보호사)
  ↓
[케어플랜 수립]
  ↓
[정기 방문 시작]
```

#### 핵심 기능
- 방문간호지시서 업로드 및 유효성 확인
- 건강 상태 입력 폼
- 간호사/요양보호사 프로필 DB
- 일정 관리 (캘린더)
- 방문 전 알림
- 가족 앱 동기화

#### 예상 소요 시간
- **4주**

### 3순위: 방문 기록 시스템 ⭐⭐⭐
**법적 필수 + 가족 모니터링 핵심 ✅**

#### 개발 이유
- **법적 요구사항**: 간호/요양 기록 의무
- **가족 안심**: 실시간 케어 내용 확인
- **품질 관리**: 서비스 품질 모니터링

#### 개발 범위

##### 3-1. 파트너 앱 (간호사/요양보호사)
```typescript
[방문 시작 체크인]
  ↓
[케어 내용 기록]
  - 간호 처치 내역
  - 생체 신호 측정 (혈압, 체온 등)
  - 재활 운동 진행
  - 특이사항 메모
  ↓
[사진 첨부] (Before/After)
  ↓
[방문 완료 체크아웃]
  ↓
[가족 앱 자동 전송]
```

##### 3-2. 가족 앱
```typescript
[방문 시작 알림] 📱
  ↓
[실시간 진행 상황]
  ↓
[방문 완료 알림]
  ↓
[기록 조회]
  - 케어 내용
  - 사진
  - 간호사 메모
```

#### 핵심 기능
- 방문 체크인/아웃
- 케어 내용 입력 폼
  - 간호 처치 체크리스트
  - 생체 신호 입력
  - 투약 확인
  - 자유 메모
- 사진 업로드
- 실시간 알림 (가족 앱)
- 기록 타임라인

#### 예상 소요 시간
- **3주**

### 4순위: 통합 돌봄 대시보드 (가족 앱) ⭐⭐
**가족 만족도 핵심 ✅**

#### 개발 이유
- **가족 안심**: 부모님 상태 한눈에 확인
- **돌봄 부담 경감**: 일일이 연락 안 해도 됨
- **서비스 가치 증명**: 플랫폼 필요성 입증

#### 개발 범위
```typescript
[가족 앱 홈]
  ↓
[부모님 통합 대시보드]
  - 오늘 일정 (방문 간호/요양)
  - 이번 주 서비스 현황
  - 최근 활동 타임라인
  - 건강 상태 요약
  - 알림 (미복약, 이상 징후 등)
```

#### 핵심 기능
- 일정 조회 (방문 간호/요양)
- 서비스 현황 (주간/월간)
- 활동 타임라인
  - 방문 간호 기록
  - 방문 요양 기록
  - 복약 기록
  - 건강 측정 기록
- 건강 상태 요약
  - 최근 혈압, 체온 등
  - 욕창/상처 진행 상황
- 알림 센터

#### 예상 소요 시간
- **2주**

### 5순위: 케어플랜 수립 시스템 ⭐⭐
**바야다 차별화 포인트 ✅**

#### 개발 이유
- **바야다 차별성**: 간호사가 전문적으로 케어플랜 설계
- **법적 근거**: 방문간호지시서 기반 체계적 케어
- **품질 관리**: 표준화된 케어 제공

#### 개발 범위
```typescript
[초기 방문 후]
  ↓
[간호사가 케어플랜 작성] (파트너 앱)
  - 건강 상태 평가
  - ADL (일상생활 능력) 평가
  - 케어 목표 설정
  - 서비스 계획 수립
  - 위험 평가 (낙상, 욕창 등)
  ↓
[케어플랜 승인] (시니어/가족)
  ↓
[케어플랜 기반 서비스 제공]
  ↓
[정기 재평가] (3개월)
```

#### 핵심 기능
- 케어플랜 작성 폼 (파트너 앱)
  - 건강 상태 평가 템플릿
  - ADL 평가 체크리스트
  - 목표 설정 (단기/장기)
  - 서비스 계획 (방문 횟수, 시간, 내용)
  - 위험도 평가
- 케어플랜 조회 (시니어/가족 앱)
- 케어플랜 승인 프로세스
- 재평가 알림 (3개월)

#### 예상 소요 시간
- **2주**

### 6순위: 시니어 앱 기본 기능 ⭐
**시니어 UX 최적화 ✅**

#### 개발 이유
- **사용자 접점**: 시니어가 직접 사용
- **자립 지원**: 가족 도움 없이도 기본 기능 사용
- **시니어 친화 UX 검증**: 실제 사용성 테스트

#### 개발 범위 (최소 기능)
```typescript
[홈 화면]
  - 오늘 일정
  - 빠른 메뉴 (전화, SOS)

[일정 조회]
  - 방문 간호/요양 일정
  - 알림

[긴급 호출]
  - SOS 버튼 (3초 길게 누르기)
  - 가족 자동 연락
  - 위치 공유

[내 정보]
  - 건강 정보
  - 케어플랜 조회
```

#### 핵심 기능
- 큼직한 UI (폰트 24px+, 버튼 60×60px)
- 단순 네비게이션 (최대 2단계)
- 음성 안내 (TTS)
- SOS 긴급 호출
- 일정 조회

#### 예상 소요 시간
- **3주**

## 🔑 Phase 1 핵심 전략

### 1. MVP로 빠르게 시작
```
완벽한 자동화 (X)
관리자 수동 매칭으로 시작 (O)

→ 빠르게 런칭 → 고객 피드백 → 개선
```

### 2. 바야다 차별화 극대화
```
간호사 + 요양보호사 초기 방문
= 일반 요양 업체와 완전히 다른 경험
```

### 3. 가족 안심 = 핵심 가치
```
가족 앱 통합 돌봄 대시보드
= 부모님 상태 실시간 확인
= 돌봄 부담 경감
```

### 4. 시니어 UX 집중
```
큼직한 UI + 단순 네비게이션 + 음성 안내
= 시니어도 쉽게 사용
```

### 5. 장기요양보험 = 안정적 수익
```
공단 부담 85% + 본인 부담 15%
= 고객 진입장벽 낮음
= 바야다는 100% 수익 확보
```
